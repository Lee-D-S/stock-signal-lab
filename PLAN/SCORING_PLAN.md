# 팩터 스코어링 설계 계획 (Factor Scoring)

## 목적

28개 스크리닝 조건을 AND 조합으로 사용하는 대신,  
각 종목이 몇 개 조건을 충족하는지 **팩터군별 득표율**로 점수화하여 매수 후보를 선정한다.

> 조건 후보는 `FACTOR_RESEARCH_PLAN.md`에서 발굴하고, 전략 성과는 `BACKTEST_PLAN.md`로 검증한다

---

## AND 방식 vs 스코어링 방식 비교

| 구분 | AND 방식 | 스코어링 방식 |
|------|----------|--------------|
| 선택 기준 | 모든 조건 동시 충족 | 일정 비율 이상 충족 |
| 신호 빈도 | 적음 (조건 많을수록 급감) | 충분 (임계값으로 조절 가능) |
| 노이즈 내성 | 약함 (하나라도 틀리면 탈락) | 강함 (일부 오신호를 다수가 상쇄) |
| 상관 지표 문제 | 없음 | 있음 → 팩터군 단위 투표로 해결 |
| 과적합 위험 | 조건 수 | 임계값 + 군별 가중치 |

---

## 1. 팩터군 구성 (5개 군)

28개 지표를 상관관계가 낮은 팩터군으로 묶어 **군 단위로 투표**한다.  
동일 군 내 지표들이 많아도 해당 군의 투표권은 1표(군 내 충족률로 계산)로 제한된다.

| 팩터군 | 지표 | 측정 대상 |
|--------|------|-----------|
| **모멘텀** | RSI, 스토캐스틱, MACD, MACD 히스토그램, OBV, OBV추세 | 단기 가격/거래량 모멘텀 |
| **추세** | MA 단기정배열(5·20·60), MA 장기정배열(60·120·240), BB위치, 거래량MA | 중장기 추세 방향 |
| **가치** | PER, PBR, EPS, BPS | 시장가 대비 저평가 여부 |
| **펀더멘털** | ROE, ROA, 영업이익률, 순이익률, 부채비율, 매출액, 영업이익, 순이익 | 기업 재무 건전성 |
| **변동성/수급** | BB 스퀴즈, BB 돌파, 피보나치 지지, 거래대금, 스토캐스틱 범위 | 변동성·수급 이상 징후 |

## 2. 스코어 계산 방식

### 단계 1 — 군 내 충족률 계산

```
군 내 충족률 = 해당 군에서 충족한 조건 수 / 해당 군 전체 조건 수

예) 모멘텀군 6개 중 4개 충족 → 군 충족률 = 4/6 = 0.67
```

### 단계 2 — 군별 가중치 적용

기본값은 균등 가중치(각 군 20%). Factor Research 결과로 IC가 높게 나온 군에 더 높은 가중치를 부여할 수 있다.

```
최종 스코어 = Σ (군 충족률 × 군 가중치)

균등 가중치 예시:
  모멘텀    0.67 × 0.20 = 0.134
  추세      1.00 × 0.20 = 0.200
  가치      0.50 × 0.20 = 0.100
  펀더멘털  0.75 × 0.20 = 0.150
  변동성    0.40 × 0.20 = 0.080
  ─────────────────────────────
  최종 스코어            0.664  (66.4%)
```

### Factor Research 연계 — IC 기반 군 가중치

```
Factor Research에서 군별 평균 IC 계산:
  모멘텀   IC 평균 0.09 → 가중치 비율 높임
  가치     IC 평균 0.06 → 중간
  펀더멘털 IC 평균 0.03 → 가중치 낮춤

군 가중치 = 군별 평균 IC / 전체 군 IC 합계
```

## 3. 임계값 설정 — 수치 통계 기반

임계값을 임의로 정하지 않고, **Train 구간 데이터에서 스코어 구간별 수익률 통계**를 뽑아 자연스러운 꺾임점(elbow)을 찾는다.

### 스코어 구간별 통계 테이블 (예시)

```
스코어 구간  종목·날짜 수  평균 20일 수익률  승률    평균 손익비
  ≥ 90%          47          +8.2%          68%       2.1
  ≥ 80%         134          +5.1%          61%       1.8
  ≥ 70%         312          +2.8%          54%       1.5
  ≥ 60%         589          +1.2%          51%       1.2
  ≥ 50%         821          +0.3%          49%       1.0
  < 50%        1203          -1.4%          43%       0.8
```

### 임계값 결정 기준

```
1. 평균 수익률 > 거래비용(왕복 0.23%) — 거래 자체가 의미있는 최저선
2. 승률 > 50% — 기댓값 플러스 구간
3. 손익비 > 1.5 — BACKTEST_PLAN.md 기준선
4. 충분한 샘플 수(> 30건) — 통계 신뢰도 확보

→ 위 조건을 동시에 만족하는 가장 낮은 스코어 = 임계값
```

### 주의: 임계값도 Train 구간에서만 결정

- Validation/Test 구간의 수익률을 보고 임계값을 조정하면 look-ahead bias 발생
- 임계값은 Train 구간 통계로 1회 결정 후 고정

## 4. 구현 모듈 구조

```
scripts/
├── scoring/
│   ├── scorer.py          # 종목별 팩터군 충족률 + 최종 스코어 계산
│   ├── threshold.py       # 스코어 구간별 수익률 통계 → 임계값 탐색
│   └── weight_tuner.py    # Factor Research IC → 군별 가중치 변환 (선택)
└── run_scoring.py         # 진입점
```

**`scorer.py`**
- screener_lib의 기존 `calc_all()` 결과를 입력으로 받음
- 조건별 True/False → 팩터군 분류 → 군 충족률 계산 → 가중 합산
- 출력: 종목별 최종 스코어 DataFrame

**`threshold.py`**
- Train 구간의 모든 (종목, 날짜) 조합에 대해 스코어 + N일 후 수익률 계산
- 스코어 구간별 승률·평균수익률·손익비·샘플 수 집계
- 테이블 출력 및 권장 임계값 제안

**`weight_tuner.py`**
- `FACTOR_RESEARCH_PLAN.md`의 analyzer.py가 산출한 IC 결과를 읽어
- 군별 평균 IC → 정규화 → 군 가중치 반환

---

## 5. 실행 예시 (완성 후)

```bash
# Train 구간에서 스코어 구간별 수익률 통계 출력 → 임계값 결정
python scripts/run_scoring.py \
  --mode threshold \
  --start 2020-01-01 --end 2022-12-31 \
  --hold-days 20

# 결정된 임계값(예: 70%)으로 오늘 매수 후보 스크리닝
python scripts/run_scoring.py \
  --mode screen \
  --threshold 0.70

# Factor Research IC 결과로 군 가중치를 조정한 스코어링
python scripts/run_scoring.py \
  --mode screen \
  --threshold 0.70 \
  --ic-weights scripts/discovery/ic_results.csv
```

## 6. 워크플로우 내 위치

```
Factor Research (FACTOR_RESEARCH_PLAN.md)
  └─ Train 구간에서 군별 IC 계산
       └─ 팩터 스코어링 (이 파일)
            ├─ IC → 군 가중치 설정
            └─ Train 구간 통계 → 임계값 결정
                 └─ Backtesting (BACKTEST_PLAN.md)
                      └─ Validation/Test 구간에서 스코어링 전략 검증
```

## 7. 구현 우선순위

```
1단계: scorer.py      — 기존 screener_lib 위에 팩터군 분류 + 스코어 계산
2단계: threshold.py   — Train 구간 통계 → 임계값 탐색 테이블
3단계: run_scoring.py — 진입점 (screen 모드)
4단계: weight_tuner   — Factor Research IC 연계 (선택 사항)
```
