# 팩터 리서치 설계 계획 (Factor Research)

## 목적

백지 상태에서 시작하여 과거 데이터가 스스로 패턴을 알려주게 하는 것.  
오른 주식과 내린 주식의 공통점을 역추적해 유망한 조건 조합 후보를 발굴한다.

> 발굴된 후보는 반드시 `BACKTEST_PLAN.md`의 Rule-based Backtest로 별도 검증할 것  
> 팩터군별 IC 결과는 `SCORING_PLAN.md`의 군 가중치 설정에 직접 활용된다

---

## Backtesting과의 차이

| 구분 | Backtesting | Factor Research |
|------|------------|-----------------|
| 시작점 | 가설 ("이 조건이 맞을 것 같다") | 백지 |
| 방향 | 조건 → 과거 검증 | 결과 → 공통점 역추적 |
| 질문 | "이 전략이 과거에 작동했는가?" | "어떤 지표가 주가 상승을 예측하는가?" |
| 결과물 | 전략의 유효성 수치 | 유망 지표 순위 + 조건 후보 |
| 위험 | 가설이 틀릴 수 있음 | 우연한 패턴 발견 위험 |

---

## 1. 동작 원리

```
1. 과거 기간에서 급등 주식 수집    (예: 20거래일 내 +20% 이상)
2. 급락 주식도 수집                (예: 20거래일 내 -15% 이하)
3. 각 주식의 급등/급락 직전 시점에서 28개 지표값 계산
4. 두 그룹 비교 → 어떤 지표가 가장 차별화되는지 분석
5. 유망 조건 후보 도출 → Backtest로 검증
```

### Look-ahead bias 방지

- 지표 계산 시 **급등/급락 시점 이전 데이터만** 사용
- 급등이 시작된 날의 종가로 지표를 계산하면 안 됨
- 기준: T일 급등 확인 → T-1일까지 데이터로 지표 계산

## 2. 분석 방법

### 방법 1 — 상관관계 분석

각 지표값과 N일 후 수익률의 상관계수 계산.

```
입력: 전체 종목 × 전체 날짜의 지표값 + 미래 수익률
출력: 지표별 상관계수 순위

예) RSI    ↔ 20일 후 수익률: 상관계수 -0.18  (RSI 낮을수록 상승)
    PBR    ↔ 20일 후 수익률: 상관계수 -0.14
    OBV    ↔ 20일 후 수익률: 상관계수  0.11
```

### 방법 2 — 그룹 평균 비교

급등/급락 그룹의 각 지표 평균값 비교.

```
입력: 급등 그룹 N건, 급락 그룹 M건
출력: 지표별 그룹 간 차이

예) RSI 평균:  급등그룹 42.3  /  급락그룹 61.8  → 차이 19.5
    PBR 평균:  급등그룹  1.2  /  급락그룹  3.1  → 차이  1.9
    ...
```

### 방법 3 — Information Coefficient (IC)

지표값 순위와 수익률 순위 간의 스피어만 상관계수.

```
IC > 0.05   유의미한 예측력
IC > 0.10   강한 예측력
IC < 0.02   노이즈 수준
```

### 방법 4 — 분포 시각화 (선택)

급등/급락 그룹별 지표 분포를 히스토그램으로 비교.  
조건 임계값 설정에 활용 (예: RSI 몇 이하일 때 급등 비율이 높은지).

## 3. 출력 예시

```
[ 팩터 리서치 결과 ]
기간: 2020-01-01 ~ 2022-12-31
급등 기준: 20거래일 +20% 이상 (N=847건)
급락 기준: 20거래일 -15% 이하 (N=612건)

지표별 예측력 (IC 기준 내림차순):
순위  지표          IC     급등그룹 평균  급락그룹 평균  해석
  1   RSI          0.12        42.3         61.8   낮을수록 상승 예측
  2   PBR          0.09         1.2          3.1   낮을수록 상승 예측
  3   OBV 추세     0.08       73% ↑        31% ↑  상승 추세일수록 좋음
  4   MA 정배열    0.07       68% 정배열   18%    정배열일수록 좋음
  5   MACD-H       0.06        +0.8         -1.2   양수일수록 좋음
  ...
 20   스토캐스틱   0.01        52.1         49.3   예측력 거의 없음

→ 추천 조건 후보:
  --rsi-max 50 --pbr-max 1.5 --obv-rising --ma-align 60,120,240
  (Backtest로 검증 필요 — 이 결과 자체가 전략이 아님)
```

---

## 4. 주의사항

- **우연한 패턴 위험**: 샘플이 적으면 통계적으로 무의미한 패턴 발견 가능
  - 급등/급락 각 그룹 30건 이상 확보 권장
- **발견 ≠ 검증**: 팩터 리서치 결과는 가설이지 검증된 전략이 아님
  - 반드시 **Train 구간에서만** 리서치하고, Validation/Test는 Backtest용으로 아껴둘 것
- **과거 데이터 한계**: 발견된 패턴이 미래에도 작동한다는 보장 없음

## 5. 구현 모듈 구조

```
scripts/
├── discovery/
│   ├── collector.py   # 급등/급락 사례 수집 + 직전 지표값 계산
│   ├── analyzer.py    # 상관관계, IC, 그룹 평균 비교 계산
│   └── report.py      # 지표별 예측력 순위 출력 + 조건 후보 제안
└── run_discovery.py   # 진입점
```

**`collector.py`**
- 전체 종목 × 전체 날짜 순회
- 각 날짜에서 N일 후 수익률 계산
- 급등/급락 기준 충족 시 직전 지표값 기록
- screener_lib의 기존 `calc_all()` 재사용

**`analyzer.py`**
- collector 결과를 받아 통계 계산
- IC, 상관계수, 그룹 평균 차이 산출
- 지표별 예측력 순위 반환

**`report.py`**
- 순위표 콘솔 출력
- CSV 저장
- 상위 N개 지표로 조건 후보 자동 조합 제안

## 6. 실행 예시

```bash
# Train 구간 기본 실행 (시가총액 상위 300개, 20일 보유)
python scripts/run_discovery.py --start 2020-01-01 --end 2022-12-31

# 단기 패턴 분석 (10일 보유, +10% 급등 기준), 상위 10개 지표 출력
python scripts/run_discovery.py \
  --start 2020-01-01 --end 2022-12-31 \
  --hold-days 10 --up-threshold 0.10 --top-n 10

# 레코드 저장 후 파라미터만 바꿔 빠르게 재분석 (API 재호출 없음)
python scripts/run_discovery.py --start 2020-01-01 --end 2022-12-31 \
  --save-records scripts/discovery/results/records.parquet

python scripts/run_discovery.py \
  --load-records scripts/discovery/results/records.parquet \
  --hold-days 10 --up-threshold 0.15 --down-threshold -0.10

# OHLCV 캐시 강제 갱신 (KIS API 재조회)
python scripts/run_discovery.py --start 2020-01-01 --end 2022-12-31 \
  --force-refresh
```

### 분석 대상 특징값 16개

| 구분 | 특징 | 해석 방향 |
|------|------|-----------|
| MA 추세 | price_vs_ma5/20/60/120/240 | 양수(+) = 이동평균 위 = 상승 우호 |
| MA 정배열 | ma_align_short, ma_align_long | 1 = 정배열 = 상승 우호 |
| MACD | macd_hist_norm, macd_cross_up | 양수 / 골든크로스 = 상승 우호 |
| 모멘텀 | rsi, stoch_k | 낮을수록 과매도 = 반등 우호 |
| 수급 | obv_rising, vol_ratio | 1 / 높을수록 상승 우호 |
| 볼린저 | bb_position, bb_squeeze | bb_position > 1 = 상단 돌파 |
| 피보나치 | fib_support | 1 = 지지선 근처 |

결과 CSV는 `scripts/discovery/results/` 에 저장된다.
IC 결과는 `SCORING_PLAN.md`의 군별 가중치 설정에 직접 활용한다.

## 7. 구현 우선순위

```
1단계: collector — 급등/급락 사례 수집 + 지표값 계산
2단계: analyzer  — IC, 상관계수, 그룹 비교 통계
3단계: report    — 순위 출력 + 조건 후보 제안
```

## 8. 권장 워크플로우

```
Factor Research (이 파일)
  └─ Train 구간에서 유망 지표 발굴
       └─ 조건 후보 조합 설계
            └─ Backtesting (BACKTEST_PLAN.md)
                 └─ Validation 구간에서 검증
                      └─ 통과 시 Test 구간으로 최종 확인
```
