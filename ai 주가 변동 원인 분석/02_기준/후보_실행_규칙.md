# 후보 실행 규칙 초안

## 기본 원칙

- 이 규칙은 확정 매매 전략이 아니라 관찰용 실행 기준이다.
- 일별 감시 후보는 가격/거래대금/DART 조건만으로 확정하지 않는다.
- 수급 재조회 후 `confirmed`인 후보만 실행 검토 대상으로 본다.
- `rejected_flow_mismatch`는 당일 제외한다.
- `pending_api_error`는 수급 확인 전이므로 매수 후보로 보지 않는다.

## H01 매수 후보

대상 조건:
- 변동성 장세
- 상승 이벤트
- 거래대금약함
- DART공시동반
- 직접반응
- 외국인기관동반매수

실행 기준:
- 다음 거래일 시가가 전일 종가 대비 과도하게 높지 않을 때만 분할 진입을 검토한다.
- 장초반 급락으로 전일 상승을 바로 되돌리면 진입을 보류한다.
- 장초반 눌림 후 전일 종가 또는 시가를 회복하면 1차 진입 후보로 본다.
- 보유 기간은 관찰 기준으로 5/10/20거래일을 모두 기록하되, 기본 비교값은 20거래일로 둔다.

위험 관리:
- 과거 백테스트 평균 수익률에 큰 아웃라이어가 포함될 수 있으므로 한 종목 비중을 크게 두지 않는다.
- 공시 내용이 악재성인데 수급만 좋을 경우 수동 검토를 우선한다.
- 다음 거래일 갭 급등 시 추격 진입하지 않는다.

## H02 반등 감시 후보

대상 조건:
- 변동성 장세
- 하락 이벤트
- 거래대금약함
- DART공시동반
- 직접반응
- 외국인기관동반매수

실행 기준:
- 하락 당일에는 원칙적으로 바로 진입하지 않는다.
- 다음 거래일 장초반 반등 확인 전까지 관찰로 둔다.
- 전일 종가 회복, 시가 회복, 또는 장중 평균가격 회복이 확인되면 단기 진입 후보로 본다.
- 반등 확인 없이 추가 하락하면 후보를 보류하고 D+5/D+10 결과만 기록한다.

위험 관리:
- H02는 외부 자료 확인 필요 이벤트가 많았으므로 뉴스/업황 필터가 붙기 전까지 자동 매수로 사용하지 않는다.
- 하락 원인이 명확한 악재 공시라면 수급이 좋아도 진입하지 않는다.
- 반등 확인 후 진입하더라도 5~10거래일 단기 관찰을 우선한다.

## 당일 후보 판정 순서

1. 일별 감시 스크립트로 후보를 만든다.
2. KIS 수급 가능 시간 이후 수급 재조회 스크립트를 실행한다.
3. `confirmed`만 관찰 로그에 추가한다.
4. 후보별 실행 규칙에 따라 다음 거래일 진입 가능성을 판단한다.
5. 실제 진입 여부와 무관하게 D+1, D+5, D+10, D+20 결과를 기록한다.
6. 최소 20건 이상 누적되면 후보별 실제 성과를 다시 검증한다.
