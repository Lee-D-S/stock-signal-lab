# 전략 세트와 관찰 방식 정리

## 1. 현재 상황 요약

현재 프로젝트는 27개 기업의 과거 주가 변동 원인을 분석해, 반복적으로 나타난 조건 조합을 전략 후보로 정리한 상태다.

분석 흐름은 아래와 같다.

```text
과거 분석 대상
27개 기업 × 20분기 = 540개 보고서

↓

이벤트 추출
6,211개 상승/하락 이벤트

↓

패턴 후보 추출
국면별 통과 패턴 6개

↓

백테스트/검토 후 유지
H01 / H02 / H03 / H06 / H04

↓

오늘 27개 종목에 적용
초기 후보 9개

↓

15:40 이후 수급 재확인
최종 통과 2개
```

즉, `H01/H02/H03/H06/H04`는 특정 종목명이 아니라 **전략 조건 세트**다.

반면 오늘 봐야 하는 2개 종목은, 그 전략 조건 세트를 오늘 27개 종목에 적용했을 때 실제로 조건을 통과한 결과다.

## 2. 전략 세트와 오늘 후보의 차이

`H01/H02/H03/H06/H04`는 필터다.

27개 종목은 검색 대상이다.

오늘 나온 2개 종목은 필터를 실제로 통과한 결과다.

```text
H01/H02/H03/H06/H04 = 필터
27개 종목 = 검색 대상
오늘 나온 2종목 = 필터 통과 결과
```

따라서 매일 봐야 하는 것은 27개 전체가 아니라, **그날 필터를 통과한 종목**이다.

오늘은 2개였지만, 내일은 0개일 수도 있고 5개일 수도 있다.

## 3. 현재 유지 중인 전략 세트

| 가설 | 용도 | 핵심 조건 |

| H01 | 매수 후보 | 변동성 장세 + 상승 + 거래대금약함 + DART공시 + 외국인기관동반매수 |
| H02 | 반등 감시 후보 | 변동성 장세 + 하락 + 거래대금약함 + DART공시 + 외국인기관동반매수 |
| H03 | 단기 반등 감시 후보 | AI/전력기기 테마장 + 하락 + 거래대금평균상회 + 수급엇갈림 + 주변공시부재 |
| H06 | 추격매수 회피 후보 | 반도체 반등장 + 상승 + 거래대금급증 + 외국인기관동반매도 + DART공시 |
| H04 | 참고 회피 후보 | 유동성/저금리 후반장 + 상승 + 거래대금급증 + 외국인기관동반매도 + DART공시 |

## 5. 20거래일 관찰의 의미

중요한 것은 **한 종목이 20거래일 동안 계속 조건을 통과하는지**가 아니다.

핵심은 **조건을 통과한 날 이후 5/10/20거래일 동안 주가가 어떻게 움직였는지**다.

즉 관찰 단위는 종목이 아니라 **신호 발생일**이다.

예시:

```text
2026-04-29 두산에너빌리티 H01 통과

↓

이 신호 1건을 기록

↓

다음 거래일 진입했다고 가정

↓

D+5, D+10, D+20 수익률 확인
```

두산에너빌리티가 이후 20일 동안 매일 H01을 계속 통과해야 하는 것은 아니다.

오히려 같은 종목이 매일 반복 통과하면 중복 신호로 봐야 한다.

## 6. 중요한 것과 중요하지 않은 것

중요하지 않은 것:

- 같은 종목이 20일 동안 계속 조건을 통과하는가
- 오늘 통과한 종목이 매일 같은 조건을 반복하는가
- 하루 신호 하나만 보고 전략을 확정하는가

중요한 것:

- 조건을 통과한 날짜 이후 수익률이 좋은가
- D+5에 반응이 나오는가
- D+10까지 반응이 유지되는가
- D+20에 최종적으로 유효한가
- 여러 종목과 여러 날짜에서 반복적으로 통하는가

## 7. 오늘 후보 2개의 관찰 방식

### 두산에너빌리티 H01

- 신호 발생일: 2026-04-29
- 가설: H01
- 유형: 매수 후보
- 관찰 방식:
  - 다음 거래일 시가 기준 진입 가능성을 본다.
  - 갭 상승이 과하면 추격하지 않는다.
  - 장초반 급락이면 보류한다.
  - D+5, D+10, D+20 수익률을 기록한다.

### 한미반도체 H02

- 신호 발생일: 2026-04-29
- 가설: H02
- 유형: 반등 감시 후보
- 관찰 방식:
  - 하락 이벤트 직후이므로 바로 매수보다 반등 확인이 우선이다.
  - 다음 거래일 전일 종가 회복, 시가 회복, 장중 평균가격 회복 여부를 본다.
  - 반등 확인 없이 추가 하락하면 진입 보류로 둔다.
  - D+5, D+10, D+20 수익률을 기록한다.

## 8. 앞으로의 운영 방식

매일 해야 할 일은 아래와 같다.

```text
1. 27개 종목에 전략 필터 적용
2. 가격/거래대금/DART 기준 후보 산출
3. KIS 수급 가능 시간 이후 수급 재조회
4. confirmed 후보만 관찰 로그에 추가
5. 신호 발생일 기준 D+1, D+5, D+10, D+20 결과 기록
6. 최소 20건 이상 신호가 쌓이면 가설별 실제 성과 재검증
```

일별 실행 명령:

```powershell
rtk python -u scripts/run_signal_research_pipeline.py --mode daily
```

수급 재조회까지 포함:

```powershell
rtk python -u scripts/run_signal_research_pipeline.py --mode daily --recheck
```

## 9. 결론

현재의 결론은 다음과 같다.

`H01/H02/H03/H06/H04`는 매일 적용할 전략 필터다.

오늘의 두 종목은 그 필터를 2026-04-29에 적용했을 때 실제로 통과한 결과다.

20거래일은 조건을 계속 만족하는지 보는 기간이 아니라, **신호 발생 후 결과를 평가하는 기간**이다.

따라서 앞으로 중요한 것은 “같은 종목이 계속 통과하는지”가 아니라, **신호가 발생한 뒤 실제로 수익이 났는지**를 계속 쌓아가는 것이다.

## 10. 전략 세트 관리 원칙

전략 세트 후보를 계속 늘리는 것이 목적은 아니다.

중요한 것은 더 많은 후보를 만드는 것이 아니라, 실제로 설명력과 재현성이 있는 후보만 남기고 나머지를 버리는 것이다.

원칙은 다음과 같다.

- 코어 전략 세트는 적게 유지한다.
- 새 후보는 시장 국면이 바뀌었거나, 기존 세트가 반복적으로 실패할 때만 추가한다.
- 새 후보는 곧바로 전략이 아니라 검증 대상이다.
- 후보 수가 많아질수록 해석이 흐려지고 신호 과다가 생기므로, 늘리는 것보다 정리하는 쪽이 우선이다.

정리하면 아래와 같다.

```text
전략세트 후보를 계속 늘리는 것 = 보조 작업
검증된 전략만 남기는 것 = 핵심 작업
```

현재 단계에서는 `H01/H02/H03/H06/H04` 같은 코어 세트를 중심으로 관찰 로그를 쌓고, 정말 다른 장세가 보일 때만 새 후보를 추가하는 방식이 맞다.
